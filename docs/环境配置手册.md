# 宿舍管理系统 — 环境配置手册

> 面向：组员从 GitHub clone 本项目到本机，搭好环境并把程序跑起来。
> 全程约 15–30 分钟（取决于是否已装 SQL Server 与 .NET SDK）。

---

## 0. 你将得到什么

一个能登录、能管理宿舍楼/班级/学生/寝室、能办理入住/调宿/退宿、能查询、能备份恢复的 WinForms 桌面程序，后端是本机 SQL Server 上的 `DormDB` 数据库。

整体结构：

```
KESHE/
  docs/   设计文档 + 实施计划 + 本手册 + 用户使用手册
  sql/    01~09 建库脚本（建数据库、表、索引、数据、视图、存储过程、触发器、验证、备份）
  src/    DormManagement   C# WinForms 三层架构应用
```

## 3. 创建数据库 `DormDB`

按 **01 → 07** 的顺序执行 `sql/` 下的脚本，就能得到一个含测试数据的可用数据库。任选一种方式：

### 方式 A：SSMS（最直观，推荐新手）

依次打开下列脚本 → 每个按 **F5** 执行，确认无报错再下一个：

```
01_create_database.sql   建库 + 完整恢复模式
02_create_tables.sql     8 张表 + 约束
03_create_indexes.sql    外键索引
04_seed_data.sql         测试数据（2 楼 / 2 寝室 / 8 床 / 2 班 / 3 学生 / admin）
05_create_views.sql      视图 v_RoomStatus、v_StudentHousing
06_stored_procedure.sql  存储过程 usp_GetRoomStudents
07_trigger.sql           触发器 trg_Allocation_Occupancy
```

### 方式 B：sqlcmd（命令行一键）

```powershell
Get-ChildItem "sql\0[1-7]_*.sql" | Sort-Object Name | ForEach-Object {
    Write-Host "执行 $($_.Name) ..."
    sqlcmd -S localhost -E -C -i $_.FullName
}
```
> `-E` Windows 认证，`-C` 信任证书。

### （可选）08、09 脚本

- `08_verify.sql`：功能与约束验证（会插入/删除测试住宿数据用于演示，**会改动数据**，非必须）。
- `09_backup_restore.sql`：备份与基于时间点恢复（PITR）演示，需要备份目录与权限（见第 6 节），非必须。

> 想重置数据库：重新执行 `01_create_database.sql` 即可（它会先删库再重建），然后再跑 02→07。

### 验证建库成功

```sql
USE DormDB;
SELECT name, recovery_model_desc FROM sys.databases WHERE name='DormDB';  -- DormDB | FULL
SELECT * FROM v_RoomStatus;  -- 应看到 101、102 两间，已住 0、空床 4、空闲
```

## 4. 配置数据库连接串 ⚠️ 关键一步

应用的连接串写在 **[AppConfig.cs](../src/DormManagement/AppConfig.cs)**：

```csharp
public const string ConnectionString =
    "Server=localhost;Database=DormDB;Trusted_Connection=True;TrustServerCertificate=True;";
```

**只有当你的 SQL Server 不是"localhost 默认实例 + Windows 认证"时才需要改它：**

| 你的情况 | 把 `Server=localhost` 改成 |
|---|---|
| 默认实例（多数情况） | `Server=localhost`（不用改） |
| 命名实例，如 Express | `Server=localhost\SQLEXPRESS` 或 `Server=.\SQLEXPRESS` |
| 远程/其他机器 | `Server=机器名或IP` |
| 用 SQL 账号而非 Windows 认证 | 去掉 `Trusted_Connection=True`，改 `User Id=sa;Password=你的密码;` |

> 改完保存即可，下次 `dotnet build/run` 生效。**连不上数据库 90% 是这一步没配对。**

## 6. 重要提示（组员务必看）

1. **备份/恢复功能里的目录是写死的绝对路径** `D:\DataBase\KESHE\backup`（见 [BackupRestoreForm.cs](../src/DormManagement/Forms/BackupRestoreForm.cs)）。
   - 如果你把项目克隆到了别的位置、或机器上**没有 D 盘**，"备份与恢复"窗体会报错或写到错误位置。
   - 解决：要么把项目放到 `D:\DataBase\KESHE`，要么修改该文件里的 `BackupDir` 常量为你本机的一个可写目录。
   - 另外，SQL Server **服务账户**需要对该备份目录有写权限；否则备份报"操作系统拒绝访问（5）"——在该文件夹"属性→安全"给 `NT Service\MSSQLSERVER` 加"修改"权限即可。
2. **不想用备份功能就别点它**，不影响其他所有功能的测试。
3. 数据库一旦执行过"时间点恢复"且中途失败，可能卡在"正在还原"状态导致登录失败；本程序已内置自动恢复，万一仍异常，在 SSMS 执行：`RESTORE DATABASE DormDB WITH RECOVERY;`。

## 7. 常见问题（FAQ）

| 现象 | 原因 / 解决 |
|---|---|
| 登录报"无法打开数据库 DormDB / 登录失败" | ① 没建库 → 执行第 3 节脚本；② 连接串不对 → 看第 4 节；③ 库卡在"正在还原" → `RESTORE DATABASE DormDB WITH RECOVERY;` |
| 启动就报连接超时/找不到服务器 | SQL Server 服务没启动；或实例名不对（命名实例要写 `localhost\实例名`） |
| `dotnet build` 报 SDK 版本错误 | 没装 .NET 10 SDK；`dotnet --version` 确认为 10.x |
| 还原包失败（NuGet） | 检查网络；或 `dotnet restore "src\DormManagement\DormManagement.csproj"` 重试 |
| 备份报"操作系统拒绝访问" | 给备份目录加 SQL Server 服务账户写权限（见第 6 节） |
| 中文显示乱码 | 确认数据库列为 `NVARCHAR`（脚本已是），源码文件为 UTF-8（仓库已是） |
